\LM\AuthAbstractor\Controller

\LM\AuthAbstractor\ControllerAuthenticationKernel

This is the main entry point of the library. It acts as an HTTP middleware.

It is initialised with the configuration of the application. It can then be passed an HTTP request among other things and it will return an HTTP response (among other things).

getAuthenticationProcessFactory()

getAuthenticationProcessFactory() : \LM\AuthAbstractor\Factory\AuthenticationProcessFactory

Returns the authentication process factory, which can be used to create authentication processes. This is the recommended way to create authentication processes, as it aims to be more backwards-compatible and is simply easier to use.

processHttpRequest()

processHttpRequest(\Psr\Http\Message\ServerRequestInterface  $httpRequest, \LM\AuthAbstractor\Model\AuthenticationProcess  $process, \LM\AuthAbstractor\Model\IAuthenticationCallback  $callback) : \LM\AuthAbstractor\Model\IAuthentifierResponse

Process an HTTP request.

This method should be called by the application's HTTP controller after having initialised a new authentication process or retrieved the current authentication process (e.g. from a PHP session).

Parameters

\Psr\Http\Message\ServerRequestInterface $httpRequest

The HTTP request.
\LM\AuthAbstractor\Model\AuthenticationProcess $process

The current authentication process.
\LM\AuthAbstractor\Model\IAuthenticationCallback $callback

A callback to execute if the authentication process fails or succeeds (e.g. if the user manages to prove their identity or not). You do not have to roll your own IAuthenticationCallback implementation and can just instantiate Callback.

Returns

\LM\AuthAbstractor\Model\IAuthentifierResponse —

An object containing an HTTP response that should be sent back to the user, and a new authentication process that should be stored to be retrieved later (e.g. in session).
